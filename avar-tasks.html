sharepoint setup
This is a one-time setup. Once saved, this config is stored in the browser and all team members on the same hosted page will share it automatically.

You will need: your SharePoint site URL, your Azure App (Client) ID, and your Tenant ID. See the setup guide below.
Azure App Registration
Found in Azure Portal → Azure Active Directory → Overview
Found in Azure Portal → App registrations → your app → Overview
SharePoint Site
The full URL of your SharePoint site (no trailing slash)
Setup Guide
Step 1 — Register an Azure App
Go to portal.azure.com → Azure Active Directory → App registrations → New registration.
Name: AVAR Tasks · Supported account types: Accounts in this org only
Redirect URI: Single-page application (SPA) → paste the URL where you will host this file

Step 2 — Add API permissions
In your app → API permissions → Add → Microsoft Graph:
 · User.Read (delegated)
 · Sites.ReadWrite.All (delegated)
Click Grant admin consent

Step 3 — Create SharePoint Lists
On your SharePoint site, create these 3 lists exactly:

List 1: TF_Users
 · Title (single line — built in, use for Name)
 · Email (single line)
 · Role (choice: superadmin, manager, member)
 · ManagerEmail (single line)
 · AzureId (single line)

List 2: TF_Tasks
 · Title (single line — built in, use for task title)
 · Description (multiple lines)
 · AssigneeEmail (single line)
 · CreatedByEmail (single line)
 · Priority (choice: high, medium, low)
 · Status (choice: open, in-progress, blocked, done)
 · DueDate (date)
 · Category (single line)

List 3: TF_Updates
 · Title (single line — built in, use for update text)
 · TaskId (single line)
 · AuthorEmail (single line)
 · UpdateType (choice: progress, blocker, note, done)

Step 4 — Host this file
Upload this HTML file to SharePoint (Site Pages or a document library) and share the link with your team. Or embed it as a Teams tab.
Sign in with your Microsoft account to access the AVAR task management portal.
